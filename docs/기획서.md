# SoccerMind 기획서 (PRD)

> 월드컵 경기를 하는 두 국가명을 입력받아 **승리 국가 · 예상 스코어 · 예상 득점자**를 예측하는, 실시간 데이터 기반 오픈소스 분석 도구.

- **문서 버전**: v1.0 (초안)
- **작성일**: 2026-06-13
- **상태**: 기획 확정, 구현 착수 전
- **관련 문서**: [아키텍처 설계](./아키텍처.md)

---

## 1. 배경 & 문제 정의

월드컵 경기 결과 예측은 대중적 관심이 높지만, 시중 서비스는 대부분 (a) 근거 없는 재미용 룰렛이거나 (b) 유료·폐쇄형 상용 분석이다. 누구나 코드를 받아 **직접 실행하고 검증할 수 있는**, 실제 데이터에 기반한 투명한 예측 도구가 부재하다.

SoccerMind는 공개된 무료 데이터(Elo 레이팅, 대표팀 스쿼드, 선수 득점 기록)와 검증된 통계 모델(Elo→포아송 스코어 매트릭스)을 결합하고, LLM으로 득점자·해설을 보강하여 **재현 가능하고 설명 가능한** 예측을 제공한다.

## 2. 목표 (Goals)

| # | 목표 | 측정 기준 |
|---|------|-----------|
| G1 | 두 국가명 입력만으로 승리국·스코어·득점자 예측 제공 | 입력→결과 1회 호출로 완결 |
| G2 | 통계적으로 일관되고 재현 가능한 결과 | 동일 입력·동일 데이터 스냅샷 → 동일 확률 출력 |
| G3 | 실제 인터넷 데이터를 실시간 수집·분석 | 일 단위 최신 데이터 + 속보 온디맨드 반영 |
| G4 | 누구나 클론하여 실행 가능한 오픈소스 | 무료 키 1개(또는 0개)로 동작, README 따라 10분 내 기동 |
| G5 | 예측 근거를 사람이 읽을 수 있게 설명 | 각 예측에 자연어 해설 첨부 |

### Non-Goals (이번 범위 밖)
- 베팅·환금성 기능 (법적 리스크, 명시적 배제)
- 클럽 리그·개별 선수 시장가치 분석
- 실시간 라이브 중계 중 인플레이 예측
- 모바일 네이티브 앱 (웹으로 충분)

## 3. 사용자 & 유스케이스

**주 사용자**: 축구 팬, 데이터/ML 학습자, 오픈소스 기여자.

- **UC-1 (예정 경기 예측)**: "브라질 vs 크로아티아" 입력 → 승리국·스코어·득점자·해설 확인.
- **UC-2 (가상 대진)**: 실제 일정에 없는 임의 두 팀("대한민국 vs 아르헨티나")도 예측. *→ 데이터 흐름은 예정 경기 조회에 의존하지 않고 두 팀을 각각 독립 조회한다.*
- **UC-3 (한국어 입력)**: "대한민국", "한국", "Korea", "South Korea" 등 어떤 표기로 입력해도 동일 팀으로 인식.
- **UC-4 (근거 확인)**: 사용자가 승률·상위 스코어라인 분포·선수별 득점 확률 등 수치 근거를 함께 확인.
- **UC-5 (재현/검증)**: 기여자가 동일 데이터 스냅샷으로 결과를 재현하고 모델 상수를 재보정.

## 4. 핵심 기능 요구사항 (Functional)

### F1. 국가명 입력 & 정규화 *(1급 요구사항)*
- 한국어·영어, 약칭·정식명, 일반적 별칭을 받아 **표준 팀 키**(ISO 3166-1 alpha-3, 예: `KOR`)로 정규화.
- 표준 키 ↔ 소스별 식별자(eloratings 문자열, football-data.org 이름, API-Football 숫자 ID, Wikipedia 문서 제목) 매핑 테이블 보유.
- 인식 실패 시 후보 제안(예: "혹시 'Korea Republic'인가요?").
- *근거: 모든 데이터 소스가 팀을 다르게 명명하므로, 정규화 부재 시 가장 조용히 깨지는 지점.*

### F2. 실시간 데이터 수집 (하이브리드)
- **구조화 API**: `football-data.org`(백본) + `API-Football`(선택적 심층).
- **랭킹/강도**: `eloratings.net` World Elo (키 불필요).
- **보강(스쿼드·결과)**: Wikipedia MediaWiki API.
- **속보(부상·출전·폼)**: 온디맨드 웹검색.
- TTL 기반 디스크 캐시로 무료 한도(100/일, 10/분) 내 운영.

### F3. 통계 예측 엔진
- Elo 레이팅 → 기대 득점(λ_A, λ_B) → **단일 스코어 매트릭스**(Dixon-Coles 보정 포아송).
- 매트릭스에서 **승/무/패 확률**과 **최빈 스코어라인**을 동시 산출(상호 불일치 불가능).
- 상위 N개 스코어라인 분포 제공.

### F4. 득점자 예측
- 선수별 기대 득점 `xG_i` = λ_team × 득점지분 × 출전가능도 + 페널티항.
- "1골 이상 득점 확률" = 1 − exp(−xG_i)로 랭킹, 팀별 상위 ~6명.

### F5. LLM 보강 (가드레일 적용)
- 입력: 통계 레이어가 산출한 구조화 JSON(λ, 승무패, 스코어라인, 후보 득점자, 컨텍스트).
- 출력: 재랭킹된 득점자 + 자연어 해설 (JSON 스키마 제약).
- **확률·스코어는 절대 변경 불가**, 스쿼드 명단에 없는 선수명 출력 시 자동 거부 후 통계 랭킹으로 폴백.

### F6. 결과 제공
- 웹 UI: 두 팀 입력 → 결과 카드(승리국, 스코어, 승무패 바, 득점자, 해설, 근거 수치).
- (선택) `GET /api/predict?team_a=...&team_b=...` JSON 엔드포인트.

## 5. 비기능 요구사항 (Non-Functional)

| 구분 | 요구사항 |
|------|----------|
| **재현성** | 동일 입력 + 동일 데이터 스냅샷 + temperature 0 → 결정적 출력 |
| **실시간성** | 변동 입력(스쿼드·부상·폼)은 일 단위 갱신 + 속보 온디맨드. *아래 §6 참조* |
| **우아한 성능 저하** | Anthropic 키 없음 → 통계 득점자 랭킹 + 템플릿 해설. API-Football 키 없음 → football-data.org + Elo로 동작 |
| **비용** | 무료 데이터 소스만 사용. 사용자별 본인 키 사용(.env), 키 미커밋 |
| **이식성** | Python 3.11+, Windows/macOS/Linux. `pip install` 또는 Docker로 기동 |
| **신뢰성** | 외부 소스 장애 시 캐시 폴백, 부분 데이터로도 예측 시도 |
| **법적/윤리** | robots.txt·Retry-After 준수, 서술적 User-Agent, 베팅 기능 배제 |

## 6. "실시간" vs 캐싱 — 설계 원칙

사용자는 **실시간 데이터 수집**을 요구하나, 무료 API 한도(100/일)는 공격적 캐싱을 강제한다. 이 긴장은 다음으로 해소한다:

> **월드컵 예측의 변동 입력은 분·초가 아니라 일 단위로 움직인다.** 스쿼드 발표, 부상 소식, 최근 폼은 하루 단위로 갱신되면 충분하다. 따라서:
> - **느린 데이터**(Elo, 스쿼드, 선수 통계): TTL 24h 캐시 → "일 단위 최신".
> - **빠른 데이터**(부상 속보, 예상 라인업, 직전 폼): **예측 시점 온디맨드 웹검색**으로 최신 반영.
>
> 이 조합이 "충분히 실시간"이며, 무료 한도와 양립한다. 사용자가 강제 새로고침을 원하면 캐시 무효화 옵션 제공.

## 7. 데이터 소스 요약

| 소스 | 용도 | 키 | 한도 |
|------|------|----|----|
| `eloratings.net` (World.tsv) | 팀 강도(Elo) | 불필요 | 비공식, 캐싱 |

| `API-Football` (api-sports.io) | WC 심층(스쿼드·득점왕) (선택) | 무료 자가발급 | 100/일 |

## 8. 예측 방법론 요약 (상세는 아키텍처 §예측 파이프라인)

```
Elo 레이팅 ──► 기대득점(λ_A, λ_B) ──► 스코어 매트릭스 M[x][y]  (← 유일한 진실의 원천)
                                          │
        ┌─────────────────────────────────┼──────────────────────────┐
        ▼                                 ▼                          ▼
   승/무/패 확률                    최빈 스코어라인              λ → 선수별 xG
  (M의 영역 합)                     (M의 argmax)               → 득점자 랭킹 → LLM 보강
```

- Elo의 유일한 임무는 λ 산출. 승무패는 Elo에서 **직접** 계산하지 않고 매트릭스에서 읽는다 → 승률과 스코어가 모순될 수 없음.
- LLM은 매트릭스 **하류**에서 득점자 랭킹과 해설만 담당, 단 하나의 확률도 못 바꾼다.

## 9. 범위 & 로드맵

### MVP (이번 사이클 목표 기능)
- F1 국가명 정규화 (한/영, 주요 32~48개국)
- F2 데이터 수집: eloratings + football-data.org + Wikipedia (캐시 포함)
- F3 통계 엔진 (Elo→λ→매트릭스→승무패·스코어라인)
- F4 통계 득점자 랭킹
- F5 LLM 보강 (있으면 사용, 없으면 폴백)
- F6 웹 UI 최소 버전 + JSON API

### 향후 (Post-MVP)
- 모델 상수 보정 파이프라인(과거 경기로 β0/β1/ρ 피팅) + 백테스트
- 예측 히스토리 저장 & 정확도 추적(브라이어 스코어/RPS)
- API-Football·TheSportsDB 통합 심화
- 토너먼트 시뮬레이션(조별리그·녹아웃 브래킷 몬테카를로)
- 다국어 UI(한/영), 결과 공유 카드

## 11. 성공 기준 (Definition of Done — MVP)

1. `대한민국 vs 브라질` 입력 → 승리국·스코어·득점자·해설 반환.
2. football-data.org 키 1개만으로 클론 후 기동 성공.
3. Anthropic 키 없이도 통계 결과 + 템플릿 해설 반환(우아한 저하).
4. 동일 데이터 스냅샷에서 결과 재현 확인.
5. README + 두 기획/설계 문서로 신규 기여자가 구조 파악 가능.
