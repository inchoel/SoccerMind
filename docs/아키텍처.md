## 3. 데이터 흐름 (예측 1회)

```
1. 입력 검증      "대한민국", "브라질"
        │
2. 정규화 (F1)    NameResolver → ("KOR", "BRA")  [+ 소스별 식별자 해석]
        │         실패 시 후보 제안 후 400 반환
        ▼
3. 데이터 수집 (F2)  각 팀을 독립 조회 (H2H는 부가정보; 대진 존재에 의존 안 함)
        │   ├─ EloProvider:        R[KOR], R[BRA]
        │   ├─ FootballData:       스쿼드, 최근결과, (가능시) H2H
        │   ├─ Wikipedia:          스쿼드/결과 보강
        │   ├─ WebSearch:          부상/라인업/폼 (온디맨드)
        │   └─ 모두 Cache 경유 (TTL: Elo·스쿼드 24h, 결과 6h)
        ▼
4. 통계 엔진 (F3)  elo.py:        δ → λ_A, λ_B  (+ 호스트면 H_host)
        │          score_matrix:  M[x][y] = DixonColes(λ_A, λ_B, ρ)
        │                         → 승무패, 최빈 스코어, 상위N 스코어라인
        ▼
5. 득점자 (F4)     scorers.py:    팀별 xG_i → P(득점≥1) 랭킹 상위 ~6명
        ▼
6. LLM 보강 (F5)   augmenter.py:  구조화 JSON 입력 → Claude(temp 0, 스키마)
        │                         → 재랭킹 득점자 + 해설
        │                         가드레일: 스쿼드 검증 실패 시 통계 랭킹 폴백
        │                         키 없으면 fallback.py 템플릿 해설
        ▼
7. 응답 조립       PredictionResponse(JSON) → UI 렌더
```

### 4.1 NameResolver (F1) — 1급 모듈

```python

ALIASES = {
  "KOR": {
    "display": "대한민국",
    "input_aliases": ["대한민국", "한국", "남한", "korea", "south korea",
                      "korea republic", "republic of korea", "kor"],
    "elo_name": "Korea South",          # eloratings 표기
    "football_data_id": 770,            # football-data.org team id
    "api_football_id": 17,              # API-Football team id
    "wikipedia": "South Korea national football team",
  },
  "BRA": { ... },
  # ... 월드컵 본선 참가국 (~48)
}
```

```python
# name_resolver.py
class NameResolver:
    def resolve(self, raw: str) -> ResolvedTeam | Ambiguous:
        key = normalize(raw)                  # 소문자, 공백/특수문자 정리, 한↔영
        if key in self._index: return self._index[key]   # 별칭 역인덱스
        cands = fuzzy_match(key, self._index, n=3)        # 실패 시 후보
        return Ambiguous(candidates=cands)
    # 각 Provider는 ResolvedTeam에서 자기 식별자(elo_name/…_id/wikipedia)를 읽음
```

- 역인덱스(별칭→표준키) 사전 구축, 퍼지 매칭(Levenshtein)으로 오타 흡수.
- **소스별 식별자 해석을 한 곳에 집중** → Provider는 팀 키만 받고 자기 식별자를 조회.

### 4.4 통계 엔진 (F3, F4)

**Elo → λ** (`elo.py`):
```python
delta = (R_a + H_a - R_b - H_b) / 400          # H: 중립=0, 호스트=+60
lam_a = exp(cfg.beta0 + cfg.beta1 * delta)     # beta0=log(T/2)≈0.262, beta1≈0.40
lam_b = exp(cfg.beta0 - cfg.beta1 * delta)     # 기하평균 총득점 보존
```

**스코어 매트릭스** (`score_matrix.py`) — Dixon-Coles 보정 포아송, **유일한 진실의 원천**:
```python
M[x][y] = poisson(x, lam_a) * poisson(y, lam_b) * dc_tau(x, y, lam_a, lam_b, rho)
M /= M.sum()                                    # τ 적용 후 재정규화 필수
# 영역 매핑 (M[x][y]=P(A가 x골, B가 y골)):
#   A승 = x>y = 하삼각(대각선 아래) = tril(M, -1).sum()
#   무  = x=y = 대각선         = trace(M)
#   B승 = x<y = 상삼각(대각선 위) = triu(M, 1).sum()
winner    = argmax(A승, 무, B승)
scoreline = unravel_index(argmax(M))            # (x, y)
```
> ⚠️ 조사 단계 의사코드의 triu/tril이 A/B 뒤바뀌어 있었음 → 위와 같이 수정 확정. (M[x][y]에서 A골=행 인덱스 x, B골=열 인덱스 y이므로 A승은 행>열 = 하삼각.)

**득점자** (`scorers.py`):
```python
xG_i = lam_team * s_i * a_i + pen_team * p_i    # s_i=득점지분, a_i=출전가능도, p_i=PK담당
P_score_i = 1 - exp(-xG_i)                       # 1골 이상 확률 → 랭킹
```

### 4.5 LLM Augmenter (F5)

- **입력 JSON**: λ, 승무패, 상위 스코어라인, 후보 득점자(xG·P), 컨텍스트(부상/폼/스쿼드).
- **출력 JSON**(스키마 강제): `refined_scorers`(스쿼드 내 재랭킹), `explanation`.
- **가드레일**:
  1. `temperature=0` — 결정적.
  2. JSON 스키마 제약 — 구조 고정.
  3. **스쿼드 멤버십 검증** — 명단에 없는 이름 출력 시 통계 랭킹으로 폴백.
  4. 시스템 프롬프트 스코프 락 — 확률·스코어는 앱이 verbatim 통과시키며 모델은 못 건드림.
- **폴백**(`fallback.py`): Anthropic 키 없으면 통계 랭킹 + 템플릿 해설(예: "{A}가 {p}% 우세, 최빈 스코어 {x}-{y}").

## 8. 예측 파이프라인 (오케스트레이터 의사코드)

```python
def predict(raw_a, raw_b, opts):
    a = resolver.resolve(raw_a); b = resolver.resolve(raw_b)
    if isinstance(a, Ambiguous) or isinstance(b, Ambiguous):
        return suggestions(a, b)                       # 400 + 후보

    ta = orchestrator.gather(a)                         # 각 팀 독립 수집 (캐시 경유)
    tb = orchestrator.gather(b)

    lam_a, lam_b = elo_to_lambda(ta.elo, tb.elo, opts.venue, cfg)
    M    = score_matrix(lam_a, lam_b, cfg.rho)
    wdl  = wdl_from_matrix(M)                            # A=tril, draw=trace, B=triu
    line = argmax_cell(M)
    sc_a = rank_scorers(lam_a, ta.squad, cfg)
    sc_b = rank_scorers(lam_b, tb.squad, cfg)

    if augmenter.available():
        refined, expl = augmenter.run(lam_a, lam_b, wdl, line, sc_a, sc_b, ctx(ta,tb))
        refined = validate_against_squads(refined, ta.squad, tb.squad) or (sc_a, sc_b)
    else:
        refined, expl = (sc_a, sc_b), fallback_explanation(wdl, line)

    return assemble_response(a, b, wdl, line, M, refined, expl, meta)
```

## 9. 테스트 전략 (요약)

- **단위**: NameResolver(별칭·퍼지·모호성), score_matrix(정규화 합=1, 영역 매핑, λ↔ΣxG 정합), elo_to_lambda(대칭성).
- **가드레일**: LLM 출력에 비-스쿼드 선수 주입 → 폴백 검증.
- **계약**: 각 Provider를 녹화된 픽스처(VCR)로 모킹 → 한도 소모 없이 테스트.
- **재현성**: 동봉 스냅샷으로 골든 예측 스냅샷 테스트.
